FAQ Bot - Документация по Batch скриптам
===========================================

Данный документ содержит описание всех batch файлов проекта FAQ Bot,
их функции, сценарии использования и инструкции по устранению неполадок.

===========================================
НОВЫЙ УНИВЕРСАЛЬНЫЙ ЗАГРУЗЧИК FAQ
===========================================

Universal_FAQ_Loader.bat (корневой каталог)
--------------------------------------------
ФУНКЦИЯ: Универсальный загрузчик файлов ВСЕХ типов в FAQ базу
ОПИСАНИЕ: Революционный инструмент для загрузки любых файлов (изображения,
         документы, таблицы, архивы, код) с автоматическим распознаванием
         и обработкой содержимого. Поддерживает OCR для изображений.
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Загрузка изображений с текстом (JPG, PNG, GIF, BMP) с помощью OCR
- Обработка документов (PDF, Word, RTF) с извлечением текста
- Импорт таблиц (CSV, Excel) с умным распознаванием структуры
- Анализ архивов (ZIP, RAR, 7Z) с извлечением содержимого
- Обработка кода (Python, JavaScript, HTML) с анализом комментариев
- Работа с любыми другими типами файлов
ВЗАИМОДЕЙСТВИЕ: 
- Создает виртуальное окружение если не существует
- Устанавливает базовые зависимости автоматически
- Запускает universal_loader/universal_faq_gui.py
- Интегрируется с существующей FAQ базой
УСТРАНЕНИЕ НЕПОЛАДОК:
- "Python не найден": Скачайте Python 3.7+ с https://python.org
- "Файл не найден": Убедитесь, что папка universal_loader/ существует
- "OCR не работает": Нажмите кнопку "Установить OCR" в программе
- "Ошибка обработки изображений": Установите Tesseract OCR отдельно
- "Не читаются PDF": Будут установлены дополнительные библиотеки
- "Большие файлы": Программа обрабатывает файлы по частям

ОСНОВНЫЕ СКРИПТЫ
===========================================

start.bat (корневой каталог)
-----------------------------
ФУНКЦИЯ: Быстрый запуск FAQ бота
ОПИСАНИЕ: Главная точка входа для запуска бота. Автоматически перенаправляет 
         выполнение на scripts/2_start_bot.bat для запуска в организованной структуре.
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ: 
- Двойной клик для быстрого запуска бота
- Запуск из командной строки для обычного использования
ВЗАИМОДЕЙСТВИЕ: Вызывает scripts/2_start_bot.bat
УСТРАНЕНИЕ НЕПОЛАДОК:
- Если скрипт не запускается, проверьте наличие scripts/2_start_bot.bat
- Убедитесь, что путь к папке scripts корректен

===========================================
СКРИПТЫ УСТАНОВКИ И НАСТРОЙКИ
===========================================

scripts/setup.bat
------------------
ФУНКЦИЯ: Полная настройка среды разработки и установка зависимостей
ОПИСАНИЕ: Создает виртуальное окружение, устанавливает все необходимые 
         зависимости, проверяет конфигурацию проекта
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Первый запуск после клонирования проекта
- Восстановление после сбоя виртуального окружения
- Установка новых зависимостей
ВЗАИМОДЕЙСТВИЕ: Независимый скрипт, создает venv/ каталог
УСТРАНЕНИЕ НЕПОЛАДОК:
- Ошибка "Python не найден": Установите Python 3.8+ и добавьте в PATH
- Ошибка создания venv: Проверьте права доступа к каталогу
- Ошибка установки пакетов: Проверьте интернет-соединение
- Отсутствие requirements.txt: Скрипт установит базовые зависимости

===========================================
СКРИПТЫ ЗАПУСКА
===========================================

scripts/2_start_bot.bat
------------------------
ФУНКЦИЯ: Основной скрипт запуска FAQ бота
ОПИСАНИЕ: Активирует виртуальное окружение, проверяет наличие run_bot.py,
         запускает бота с обработкой ошибок
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Основной способ запуска бота в продакшене
- Запуск после изменения конфигурации
- Плановый перезапуск бота
ВЗАИМОДЕЙСТВИЕ: Может быть вызван из start.bat или напрямую
УСТРАНЕНИЕ НЕПОЛАДОК:
- "Файл run_bot.py не найден": Проверьте структуру проекта
- Ошибки импорта: Запустите setup.bat для переустановки зависимостей
- Ошибка токена: Проверьте файл .env и токен бота

scripts/restart_bot.bat
------------------------
ФУНКЦИЯ: Перезапуск бота с принудительным завершением процессов
ОПИСАНИЕ: Останавливает все процессы Python и перезапускает бота
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Когда бот завис или не отвечает
- После критических изменений в коде
- Экстренная перезагрузка
ВЗАИМОДЕЙСТВИЕ: Использует taskkill для остановки Python процессов, 
                затем вызывает 2_start_bot.bat
УСТРАНЕНИЕ НЕПОЛАДОК:
- Процесс не завершается: Перезагрузите компьютер
- Ошибка прав доступа: Запустите как администратор

===========================================
СКРИПТЫ ОЧИСТКИ И ОБСЛУЖИВАНИЯ
===========================================

scripts/1_clean_all.bat
------------------------
ФУНКЦИЯ: Полная очистка проекта от временных файлов разработки
ОПИСАНИЕ: Удаляет кэш, тестовые файлы, отчеты разработки, сбрасывает БД
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Подготовка к чистому запуску
- Очистка после разработки/тестирования
- Устранение проблем с кэшем
ВЗАИМОДЕЙСТВИЕ: Вызывает clear_cache.bat и reset_db.bat
УСТРАНЕНИЕ НЕПОЛАДОК:
- Файлы не удаляются: Закройте все программы, использующие файлы
- Ошибка прав доступа: Запустите как администратор

scripts/clear_cache.bat
------------------------
ФУНКЦИЯ: Очистка кэша эмбеддингов и индексов FAISS
ОПИСАНИЕ: Удаляет файлы кэша (*.pkl, *.faiss), Python кэш (__pycache__)
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- После изменения faq.json
- При проблемах с поиском
- После обновления модели эмбеддингов
ВЗАИМОДЕЙСТВИЕ: Может быть вызван из 1_clean_all.bat или независимо
УСТРАНЕНИЕ НЕПОЛАДОК:
- Файлы кэша не удаляются: Остановите бота перед очисткой
- __pycache__ остается: Проверьте права доступа к каталогам

scripts/reset_db.bat
---------------------
ФУНКЦИЯ: Сброс базы данных аналитики
ОПИСАНИЕ: Удаляет файл аналитической БД для начала с чистого листа
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Сброс статистики использования
- Очистка логов взаимодействий
- Подготовка к новому периоду мониторинга
ВЗАИМОДЕЙСТВИЕ: Вызывается из 1_clean_all.bat или независимо
УСТРАНЕНИЕ НЕПОЛАДОК:
- БД не удаляется: Остановите бота и убедитесь, что нет блокировок файла

===========================================
СКРИПТЫ ЗАГРУЗКИ ДАННЫХ
===========================================

scripts/3_bulk_loader.bat
--------------------------
ФУНКЦИЯ: Массовая загрузка FAQ данных из Excel файлов
ОПИСАНИЕ: Запускает GUI интерфейс для загрузки вопросов и ответов из файлов
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Массовое обновление FAQ базы
- Импорт данных из внешних источников
- Управление большими объемами FAQ данных
ВЗАИМОДЕЙСТВИЕ: Требует установленные зависимости pandas, openpyxl, tkinter
УСТРАНЕНИЕ НЕПОЛАДОК:
- Ошибки импорта библиотек: Запустите setup.bat
- GUI не открывается: Проверьте поддержку tkinter в системе
- Ошибки чтения Excel: Проверьте формат и структуру файла

scripts/4_load_files.bat
-------------------------
ФУНКЦИЯ: Загрузка файлов в систему с множественными интерфейсами
ОПИСАНИЕ: Предоставляет drag-and-drop, GUI и командную строку для загрузки
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Загрузка документов в файловую систему бота
- Обновление ресурсов бота
- Массовая загрузка файлов различных типов
ВЗАИМОДЕЙСТВИЕ: Может работать с tkinterdnd2 для drag-and-drop
УСТРАНЕНИЕ НЕПОЛАДОК:
- Drag-and-drop не работает: Проверьте установку tkinterdnd2
- Большие файлы не загружаются: Проверьте лимит 50MB
- Ошибки разрешений: Проверьте права доступа к папке files/

===========================================
СКРИПТЫ ОБНОВЛЕНИЯ
===========================================

scripts/update_bot.bat
-----------------------
ФУНКЦИЯ: Обновление кода бота и очистка кэша
ОПИСАНИЕ: Подготавливает проект к применению обновлений
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- После получения обновлений кода
- Перед применением новой версии
- Подготовка к изменениям в структуре
ВЗАИМОДЕЙСТВИЕ: Базовый скрипт, может быть расширен для автоматических обновлений
УСТРАНЕНИЕ НЕПОЛАДОК:
- Скрипт слишком простой: Может потребоваться ручная доработка
- Обновления не применяются: Проверьте процесс получения новой версии

scripts/upgrade_embeddings.bat
-------------------------------
ФУНКЦИЯ: Обновление модели эмбеддингов на более современную
ОПИСАНИЕ: Автоматизированное переключение на BGE-M3 модель с сохранением 
         старой версии и тестированием производительности
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- Улучшение качества поиска FAQ
- Переход на более современные модели
- Тестирование новых подходов к эмбеддингам
ВЗАИМОДЕЙСТВИЕ: Создает бэкапы, тестирует производительность, 
                взаимодействует с clear_cache.bat
УСТРАНЕНИЕ НЕПОЛАДОК:
- Ошибка загрузки модели: Проверьте интернет-соединение
- Проблемы производительности: Откатитесь к бэкапу
- Ошибки тестирования: Проверьте наличие тестовых данных

scripts/create_shortcut.bat
----------------------------
ФУНКЦИЯ: Создание ярлыка на рабочем столе для быстрого запуска
ОПИСАНИЕ: Автоматически создает ярлык start.bat на рабочем столе
СЦЕНАРИЙ ИСПОЛЬЗОВАНИЯ:
- После установки для удобства пользователя
- Создание быстрого доступа к боту
ВЗАИМОДЕЙСТВИЕ: Независимый утилитарный скрипт
УСТРАНЕНИЕ НЕПОЛАДОК:
- Ярлык не создается: Проверьте права доступа к рабочему столу
- Неверный путь: Убедитесь, что скрипт запущен из правильной папки

===========================================
ОБЩИЕ РЕКОМЕНДАЦИИ
===========================================

ПОРЯДОК ПЕРВОГО ЗАПУСКА:
1. Запустите setup.bat для установки окружения
2. Настройте .env файл с токеном бота
3. Проверьте наличие faq.json
4. Запустите start.bat для проверки работы
5. При необходимости используйте create_shortcut.bat

РЕГУЛЯРНОЕ ОБСЛУЖИВАНИЕ:
- Используйте clear_cache.bat после изменения FAQ
- Запускайте 1_clean_all.bat для полной очистки
- Применяйте upgrade_embeddings.bat для улучшения качества

УСТРАНЕНИЕ ОБЩИХ ПРОБЛЕМ:
- Ошибки кодировки: Все скрипты используют UTF-8 (chcp 65001)
- Проблемы путей: Скрипты автоматически устанавливают рабочий каталог
- Ошибки прав: Запускайте от имени администратора при необходимости
- Проблемы зависимостей: Переустановите окружение через setup.bat

===========================================
СТРУКТУРА ВЗАИМОДЕЙСТВИЯ СКРИПТОВ
===========================================

start.bat → scripts/2_start_bot.bat
1_clean_all.bat → clear_cache.bat + reset_db.bat
setup.bat (независимый)
3_bulk_loader.bat (независимый)
4_load_files.bat (независимый)
restart_bot.bat → 2_start_bot.bat
upgrade_embeddings.bat → clear_cache.bat

===========================================
ТЕХНИЧЕСКАЯ ПОДДЕРЖКА
===========================================

Если проблема не решается:
1. Проверьте журналы в папке logs/
2. Запустите 1_clean_all.bat для полной очистки
3. Переустановите окружение через setup.bat
4. Убедитесь в корректности .env и faq.json
5. Проверьте наличие всех файлов проекта

Все скрипты разработаны с учетом безопасности и включают проверки ошибок.
При возникновении критических проблем рекомендуется полная переустановка
через setup.bat.
