# FAQ Bot - Telegram Bot с векторным поиском

## Описание проекта
Это Telegram бот для ответов на часто задаваемые вопросы (FAQ) с использованием векторного поиска.
Бот использует модель sentence-transformers для создания эмбеддингов и FAISS для быстрого поиска.

## Структура проекта

### Основные модули:
- `main.py` - Точка входа в приложение
- `config.py` - Конфигурация бота
- `database.py` - Работа с базой данных аналитики
- `faq_loader.py` - Загрузка FAQ и создание векторных индексов
- `handlers.py` - Обработчики сообщений бота
- `middlewares.py` - Middleware для внедрения зависимостей
- `faq.json` - База знаний FAQ
- `.env` - Переменные окружения (токен бота)

## Утилиты и BAT-файлы

### 🚀 Основные скрипты запуска:

**`setup.bat`** 🔧
- ПЕРВЫЙ ЗАПУСК: Установка и настройка проекта
- Проверяет наличие Python и pip
- Создает виртуальное окружение (venv)
- Устанавливает все зависимости из requirements.txt
- Проверяет наличие необходимых файлов (.env, faq.json, config.py)
- РЕКОМЕНДУЕТСЯ запускать при первом использовании проекта

**`2_start_bot.bat`** 🤖
- ЗАПУСК БОТА: Основной скрипт для запуска бота
- Автоматически активирует виртуальное окружение (если есть)
- Проверяет наличие main.py перед запуском
- Показывает статус выполнения и ошибки
- Используйте для ежедневного запуска бота

**`3_bulk_loader.bat`** 📁
- МАССОВАЯ ЗАГРУЗКА: Загрузка данных в FAQ из различных источников
- Поддерживает CSV, Excel, JSON файлы и папки с файлами
- Предоставляет выбор между GUI и командной строкой
- Автоматическое создание шаблонов
- ИСПОЛЬЗУЙТЕ: Для загрузки больших объемов данных в FAQ

**`4_load_files.bat`** 📂
- ЗАГРУЗЧИК ФАЙЛОВ: Универсальный интерфейс для загрузки файлов в FAQ
- Объединяет все методы загрузки в одном удобном интерфейсе
- 4 способа загрузки: GUI, Drag&Drop, простой загрузчик, командная строка
- Встроенная диагностика и справочная система
- Автоматическая проверка зависимостей и состояния системы
- Пошаговые инструкции для новичков
- РЕКОМЕНДУЕТСЯ: Основной инструмент для загрузки файлов в FAQ

### 🧹 Скрипты очистки:

**`clear_cache.bat`** 🗑️
- ОЧИСТКА КЭША: Удаляет временные файлы и кэш
- Удаляет файлы эмбеддингов (faq_embeddings.pkl)
- Удаляет индексы FAISS (faq_index.faiss)
- Очищает Python кэш (__pycache__)
- Удаляет временные базы данных (temp_*.db, test_*.db)
- Очищает файлы логов (bot.log, logs/bot.log)
- Удаляет файлы результатов тестов
- ИСПОЛЬЗУЙТЕ: После изменения faq.json или при проблемах с поиском

**`reset_db.bat`** 📊
- СБРОС АНАЛИТИКИ: Удаляет базу данных аналитики
- Удаляет analytics.db (статистика использования бота)
- ИСПОЛЬЗУЙТЕ: Для сброса статистики или при проблемах с базой данных

**`restart_bot.bat`** 🔄
- **ПЕРЕЗАПУСК БОТА**: Полный перезапуск с диагностикой
- Корректно останавливает текущие процессы бота
- Запускает диагностику состояния
- Перезапускает бота с новыми настройками
- **ИСПОЛЬЗУЙТЕ**: Когда бот перестал отвечать или работает нестабильно

**`clean_all.bat`** 🧹
- ПОЛНАЯ ОЧИСТКА РАЗРАБОТКИ: Удаляет ВСЕ временные файлы
- Вызывает clear_cache.bat для основной очистки
- Удаляет файлы разработки (test_*.py, demo_*.py)
- Удаляет отчеты и документацию разработки (*.md файлы)
- Вызывает reset_db.bat для сброса аналитики
- ИСПОЛЬЗУЙТЕ: Для подготовки проекта к продакшену или чистому перезапуску

### 🔄 Скрипты обновления:

**`update_bot.bat`** 🔄
- ПОЛНОЕ ОБНОВЛЕНИЕ: Комплексное обновление проекта
- Последовательно вызывает clear_cache.bat и reset_db.bat
- Обеспечивает чистое состояние проекта
- ИСПОЛЬЗУЙТЕ: После обновления кода или изменения конфигурации

## Рекомендуемый порядок использования

### При первой установке:
1. `setup.bat` - установка и настройка
2. Настройте .env файл с токеном бота
3. `4_load_files.bat` - загрузите начальные данные в FAQ
4. `2_start_bot.bat` - запуск бота

### При ежедневном использовании:
- `2_start_bot.bat` - обычный запуск бота
- `4_load_files.bat` - добавление новых файлов в FAQ

### При загрузке новых данных в FAQ:
1. `4_load_files.bat` - универсальный загрузчик файлов
2. Выберите подходящий метод загрузки (GUI рекомендуется)
3. `clear_cache.bat` - очистка кэша (если нужно)
4. `2_start_bot.bat` - перезапуск бота

### При изменении FAQ или конфигурации:
1. `clear_cache.bat` - очистка кэша
2. `start_bot.bat` - запуск бота

### При проблемах или обновлении:
1. `update_bot.bat` - полное обновление
2. `start_bot.bat` - запуск бота

### Для подготовки к продакшену:
1. `clean_all.bat` - полная очистка
2. `start_bot.bat` - запуск бота

## Особенности работы бота

### 🔒 Система аутентификации (НОВОЕ в v2.0)
Бот теперь защищен системой двухфакторной аутентификации для обеспечения доступа только сотрудникам:
- **Первый запуск**: Новые пользователи должны ввести пароль "1337" после команды /start
- **Постоянный доступ**: После успешной аутентификации пользователь получает постоянный доступ
- **Админ-доступ**: Администратор (ADMIN_ID) не нуждается в аутентификации
- **Защита всех функций**: Все команды и callback queries защищены аутентификацией
- **Мониторинг**: Ведется полная статистика аутентифицированных пользователей

### 🚀 Автоматическая отправка файлов
Бот автоматически отправляет файлы без подтверждения в следующих случаях:
- **Один файл**: Например, запрос "Fold7" мгновенно отправляет маркетинговые вставки
- **Одна ссылка**: Например, запрос "Рассрочка" сразу открывает инструкцию

Для сложных случаев (несколько файлов или ресурсов) показывается клавиатура для выбора.

### 🔧 Диагностика и мониторинг
Бот теперь имеет улучшенные возможности диагностики и самовосстановления:
- **Автоматическое переподключение**: При обрыве связи бот автоматически переподключается к Telegram
- **Проверка состояния**: Каждые 5 минут проверяется состояние подключения
- **Диагностика проблем**: `python diagnose_bot.py` - анализ логов и проверка подключения
- **Улучшенное логирование**: Подробные логи сетевых операций и ошибок

### 🔍 Семантический поиск
Бот использует модель sentence-transformers для понимания смысла вопросов:
- Порог сходства: 0.7 (настраивается в config.py)
- Поддержка вариаций запросов в faq.json
- Кэширование эмбеддингов для скорости

### 📊 Аналитика и логирование
- Статистика запросов и успешных ответов
- Логирование неотвеченных вопросов
- Отслеживание запрещенных слов
- Экспорт статистики в файл

## Установка зависимостей
```bash
pip install sentence-transformers aiogram python-dotenv faiss-cpu
```

Или используйте setup.bat для автоматической установки.

## Настройка
1. Создайте файл `.env` в корне проекта:
```
BOT_TOKEN=your_telegram_bot_token
ADMIN_ID=your_telegram_user_id
```

2. Отредактируйте `faq.json` согласно вашим потребностям

## Файлы, которые создаются автоматически
- `faq_embeddings.pkl` - кэш эмбеддингов (удаляется при clear_cache.bat)
- `faq_index.faiss` - индекс FAISS для поиска (удаляется при clear_cache.bat)
- `analytics.db` - база данных статистики (удаляется при reset_db.bat)
- `bot.log` - файл логов (удаляется при clear_cache.bat)
- `__pycache__/` - кэш Python (удаляется при clear_cache.bat)

## Устранение проблем

### Проблемы с аутентификацией:
1. **Пользователь вводит правильный пароль, но бот не пускает**: Проблема исправлена в v2.0 - middleware был отключен
2. Пользователь ввёл неверный пароль - попросите ввести его заново
3. Нужно сбросить всех пользователей - запустите reset_db.bat
4. Смена пароля - измените ACCESS_PASSWORD в config.py
5. Просмотр списка пользователей - /auth_users (для админа)

### Бот перестал отвечать (главная проблема):
1. **Немедленное решение**: Запустите `restart_bot.bat`
2. **Диагностика**: Запустите `python diagnose_bot.py`
3. **Проверьте логи**: Откройте `bot.log` и найдите последние ошибки
4. **Сетевые проблемы**: Новая версия автоматически переподключается
5. **Профилактика**: Перезапускайте бота раз в сутки

### Бот не запускается:
1. Запустите `setup.bat` для проверки зависимостей
2. Проверьте токен в .env файле
3. Используйте `clear_cache.bat` и попробуйте снова

### Бот неправильно отвечает на вопросы:
1. Проверьте содержимое faq.json
2. Запустите `clear_cache.bat` для пересоздания индексов
3. Перезапустите бота

### Проблемы с загрузкой файлов:
1. **GUI интерфейс не запускается**: Проверьте установку tkinter (входит в Python)
2. **Drag&Drop не работает**: Установите tkinterdnd2 (`pip install tkinterdnd2`)
3. **Excel файлы не читаются**: Установите openpyxl (`pip install openpyxl`)
4. **Ошибки кодировки**: Сохраняйте файлы в UTF-8 кодировке
5. **Файлы не находятся**: Проверьте путь и доступ к файлам
6. **Нет результата**: Используйте "Просмотр FAQ" для проверки загруженных данных
7. **Системные проблемы**: Запустите диагностику через `4_load_files.bat` (пункт 7)
8. **Нужна помощь**: Откройте справку в `4_load_files.bat` (пункт 8)
